任务4：卷积核可视化：
<img width="1000" height="500" alt="image" src="https://github.com/user-attachments/assets/0897cb23-4f8e-463e-90e1-50dfafd5221f" />

1.卷积核特征分析：
训练完成后，这些 3×3 的卷积核明显呈现出不同的边缘、方向和纹理特征：
边缘检测：部分卷积核的黑白 / 明暗对比明显，专门用来识别数字的轮廓边缘。
方向特征：有的卷积核侧重提取水平方向的边缘，有的侧重垂直方向，还有的则对斜向边缘敏感。
局部模式：部分卷积核会对拐角、交叉点、线段端点等局部结构有强响应，这正是识别手写数字所需的关键特征。
这些特征说明，网络在训练过程中，已经自动学习到了识别手写数字所需的基础视觉模式。

2.卷积核的训练过程说明
卷积核的参数是通过反向传播算法在训练过程中自动学习得到的：
初始化：卷积核的初始权重是随机的，此时它对任何特征都没有特定的偏好。
前向传播：用随机初始化的卷积核对输入图像进行卷积操作，提取出初始的特征图，再通过后续层计算出预测结果。
误差计算：将预测结果与真实标签对比，计算损失值。
反向传播与更新：根据损失值，通过链式法则反向计算每个卷积核参数对损失的贡献（梯度），再用优化器（如 Adam）更新卷积核的权重，使损失值不断降低。
迭代优化：这个过程不断重复，卷积核的权重被逐步调整，最终形成能有效提取特定边缘、方向和纹理特征的模式，以完成数字分类任务。

任务5：Feature map 可视化
<img width="1000" height="500" alt="image" src="https://github.com/user-attachments/assets/34e16aa5-6eed-458f-ba67-a791a7f3b424" />

1.特征图响应分析
从图中可以清晰看到，不同的特征图对图像的不同区域表现出了不同的响应强度：
对边缘轮廓的响应：大部分特征图都对数字 “7” 的边缘和轮廓区域有强响应，尤其是顶部的水平笔画和右侧的斜向笔画，在部分图中呈现为亮白色区域，而背景区域则较暗。
对局部结构的响应：部分特征图主要对数字的水平笔画（顶部横杠）有强烈响应，而对斜向笔画的响应较弱；另一部分特征图则更关注斜向笔画的轮廓。
对纹理细节的响应：少数特征图对数字的拐角、端点等局部细节有突出响应，这些是区分不同数字的关键信息。

2.卷积核与特征提取说明
这些不同的响应模式，正是由不同的卷积核提取不同图像特征导致的：
每张特征图对应一个特定的卷积核，该卷积核被训练为专门提取某一类视觉特征。
例如，一个对水平边缘敏感的卷积核，会在数字 “7” 的水平笔画位置产生高响应；而一个对斜向边缘敏感的卷积核，则会在斜向笔画位置产生高响应。
第一层卷积核主要提取低级视觉特征，如边缘、方向和局部纹理，后续网络层则会在此基础上组合这些低级特征，形成更复杂的高级特征（如整体形状），最终完成数字分类。

任务6：错误分类样本分析
<img width="1200" height="400" alt="image" src="https://github.com/user-attachments/assets/a03a960f-73dc-4294-9b46-3e9b963848ee" />

1.最容易被混淆的类别：
从错误样本可以看出，以下几对数字最容易被模型混淆：
3 和 8：真实为 3，预测为 8；两者轮廓高度相似，仅局部细节有差异。
6 和 0：真实为 6，预测为 0；手写体中 6 的下半圈闭合不明显时，很容易被误判为 0。
2 和 7：真实为 2，预测为 7；当 2 的底部弯钩不明显时，整体形态与 7 非常接近。
5 和 3、5 和 9：真实为 5，分别预测为 3 和 9；手写 5 的形状变化较大，容易和 3、9 混淆。
8 和 6：真实为 8，预测为 6；8 的上半部分不明显时，会被误判为 6。
整体来看，形状相似、局部特征模糊的数字（如 3/8、6/0、2/7、5/3/9） 是模型最容易出错的类别。

2. 错误原因分析
这些错误的主要原因可以归结为以下几点：
手写体形态的多样性：部分数字的手写风格差异大，导致类内差异大、类间差异小，如变形的 “2” 和 “7”、连笔的 “3” 和 “8”。
局部特征缺失或不清晰：部分样本的关键特征（如 3 的开口、8 的中间横线、6 的底部弯钩）不明显，模型难以提取有效区分特征。
模型容量有限：本次实验使用的简易 CNN 模型层数较少，对复杂、模糊样本的区分能力有限。

3. 准确率提升改进方案
针对以上问题，可以从数据、模型和训练方法三个方面进行改进：
（1）数据层面
数据增强：对训练集进行旋转、平移、缩放、添加噪声等操作，增加模型对不同形态手写数字的鲁棒性。
过采样易错类别：对容易混淆的数字（如 3、5、8）增加训练样本数量，让模型更关注这些类别的特征。
（2）模型结构层面
加深网络：增加卷积层和池化层的数量，提升模型对复杂特征的提取能力。
引入正则化：添加 Dropout 层、Batch Normalization 层，防止过拟合，提升泛化能力。
（3）训练方法层面
调整超参数：优化学习率、批次大小，或使用学习率衰减策略，让模型收敛更稳定。
使用更优优化器：或调整优化器参数（如 Adam 的权重衰减），提升训练效果。
增加训练轮数：在不发生过拟合的前提下，适当增加训练轮数，让模型更充分地学习。

任务7：混淆矩阵
<img width="1000" height="800" alt="image" src="https://github.com/user-attachments/assets/94bbfc80-987f-44cb-8737-54d9fe3520d1" />

1. 对角线元素的含义
混淆矩阵的对角线元素，表示模型预测正确的样本数量：
行表示真实类别，列表示预测类别
对角线位置（真实类别 = 预测类别）上的数值，就是该类别被正确分类的样本数
数值越大，说明模型对该类别的识别准确率越高
从图中可以看到，对角线元素（如 0:975、1:1130、7:1006 等）数值都很大，说明模型整体分类效果良好

2. 非对角线元素的含义
混淆矩阵的非对角线元素，表示模型预测错误的样本数量：
非对角线元素的数值越大，说明这两个类别之间越容易被模型混淆

3. 混淆最严重的类别
从混淆矩阵中可以看出，最严重的混淆主要集中在 5 和其他数字之间：
真实为 5 的样本中，有 7 个被预测为 3，7 个被预测为 9，是所有类别中错误总数最多的
其次是真实为 7 的样本中，有 10 个被预测为 9，也是较明显的混淆
因此，5 与 3、5 与 9，混淆很严重，7 与 9 之间的混淆最为严重，这也与我们在错误样本中观察到的现象一致，这类数字的手写形态相似度较高，模型容易产生误判。
